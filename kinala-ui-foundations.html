Accessibility Audit

✓

lang attribute

Document has lang="en" — screen readers use correct pronunciation.

✓

Heading hierarchy

Page uses semantic <h1>/<h2> elements, not styled <p> tags.

✓

Form labels

Every input has an associated <label for>. No placeholder-only labelling.

✓

Focus indicators

Visible :focus-visible ring on all interactive elements. Mouse users are unaffected.

✓

Button semantics

All buttons use <button type="button">. Decorative buttons have aria-hidden.

✓

Error messaging

Error inputs use aria-invalid + aria-describedby pointing to a role="alert" message.

✓

Decorative SVGs

Logo SVG has aria-hidden="true" focusable="false". Hero wrapper carries the label.

⚠

foreground.muted contrast

#756858 on #FBF8F3 is ~3.5:1 — fails WCAG AA (4.5:1) for body text. Consider using sand-700 #5A4F44 (~5.2:1) for text below 18px.

⚠

caption color contrast

12px caption text is below the 18px threshold — it needs a 4.5:1 ratio. #5A4F44 on #FBF8F3 passes at ~5.2:1.
